🎓 Training Center Platform

«Обучающий микросервисный проект для изучения современной Java Backend-разработки.»

О проекте

Training Center Platform — это учебная платформа, разрабатываемая в формате командной разработки с использованием современных технологий Java Backend.

Главная цель проекта — максимально приблизить процесс разработки к работе в коммерческой компании. Участники не просто пишут код, а работают с Git, GitHub, Jira, Code Review, Pull Request, Docker, PostgreSQL и другими инструментами, которые ежедневно используются в реальных проектах.

Проект развивается поэтапно: от создания простых CRUD-сервисов до полноценной микросервисной архитектуры с обменом сообщениями, безопасностью, мониторингом и контейнеризацией.

Архитектурный подход

Каждый микросервис отвечает только за свою предметную область и имеет собственную базу данных.

Сервисы взаимодействуют между собой через REST API, а в дальнейшем — через Apache Kafka.

Такой подход позволяет:

- уменьшить связанность сервисов;
- независимо развивать каждый сервис;
- масштабировать систему;
- использовать современные подходы Enterprise-разработки.

🏗 Архитектура проекта

Общая структура проекта

Проект построен по принципам микросервисной архитектуры.

Каждый сервис является самостоятельным Spring Boot приложением, имеет собственную бизнес-логику и собственную базу данных.

Это позволяет:

- независимо развивать каждый сервис;
- уменьшить связанность компонентов;
- масштабировать систему;
- разделять ответственность между командами разработки.

Описание сервисов

User Service

Отвечает за работу с пользователями.

Функциональность:

- регистрация пользователей;
- получение информации о пользователе;
- изменение данных пользователя;
- удаление пользователя;
- поиск пользователей.

База данных:

user_db

---

Course Service

Отвечает за управление курсами.

Функциональность:

- создание курса;
- изменение курса;
- удаление курса;
- получение информации о курсах.

База данных:

course_db

---

Enrollment Service

Отвечает за запись студентов на курсы.

Функциональность:

- запись на курс;
- просмотр записей;
- изменение записи;
- удаление записи.

База данных:

enrollment_db

---

Notification Service

Отвечает за отправку уведомлений.

Планируемая функциональность:

- Email уведомления;
- получение событий из Kafka;
- шаблоны уведомлений;
- журнал отправленных сообщений.

---

Auth Service

Отвечает за аутентификацию и авторизацию пользователей.

Планируемая функциональность:

- регистрация;
- вход в систему;
- JWT;
- Refresh Token;
- управление ролями.

---

Payment Service

Отвечает за обработку платежей.

Планируемая функциональность:

- создание платежей;
- изменение статуса платежей;
- история операций;
- интеграция с внешними платёжными системами.

---

File Service

Отвечает за работу с файлами.

Планируемая функциональность:

- загрузка файлов;
- скачивание файлов;
- хранение файлов;
- интеграция с MinIO.

---

API Gateway

Единая точка входа в систему.

Будет выполнять:

- маршрутизацию запросов;
- проверку JWT;
- фильтрацию запросов;
- централизованную обработку безопасности.

---

Discovery Service

Сервис регистрации и обнаружения микросервисов.

Планируется использование:

- Spring Cloud Netflix Eureka.

---

Config Server

Централизованное хранение конфигурации всех микросервисов.

Будет добавлен на одном из последних этапов разработки.

---

Базы данных

Каждый микросервис использует собственную базу данных.

User Service        → user_db

Course Service      → course_db

Enrollment Service  → enrollment_db

Такой подход соответствует принципам микросервисной архитектуры и исключает прямой доступ сервисов к данным друг друга.

---

Взаимодействие сервисов

На начальном этапе сервисы работают независимо.

По мере развития проекта будет реализовано взаимодействие через:

- REST API;
- Spring Cloud OpenFeign;
- Apache Kafka.

Это позволит познакомиться с синхронным и асинхронным взаимодействием между микросервисами.
